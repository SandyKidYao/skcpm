# 文档一致性检查（只读模式）

检查项目文档与代码的一致性，输出详细的差异报告。**此技能只读不写，不修改任何文件。** 适合在更新前先预览差异，或定期审查文档健康度。

## 工作流程

### 第一步：确定目标文档

根据 `$ARGUMENTS` 决定要检查的文档：

**有参数时**：直接使用用户指定的文件路径。

**无参数时**：自动扫描项目，查找以下类型的文档文件：
- 根目录：`README.md`、`CLAUDE.md`、`CONTRIBUTING.md`、`CHANGELOG.md`
- 文档目录：`docs/`、`doc/`、`documentation/` 下的 `.md` 文件
- 部署相关：包含 deploy、setup、install 等关键词的文档
- 接口文档：包含 api、interface、endpoint 等关键词的文档
- AI 开发指导：`CLAUDE.md`、`.cursorrules`、`.github/copilot-instructions.md` 等

使用 Glob 工具扫描，列出找到的文档文件清单，让用户确认要检查哪些文档。

### 第二步：逐份文档分析

对每份目标文档执行以下流程：

1. **阅读文档全文**：使用 Read 工具读取文档内容，理解文档描述的主题和范围。

2. **提取关键断言**：识别文档中对代码行为的具体描述，包括但不限于：
   - 文件路径和目录结构的描述
   - 函数、类、接口的名称和用法
   - 配置项、环境变量的说明
   - 命令行用法和参数说明
   - 架构和数据流的描述
   - 依赖和版本要求
   - 部署步骤和前置条件

3. **对照代码验证**：对每个关键断言，使用 Grep、Glob、Read 工具在代码中验证其准确性。

4. **记录差异**：记录每个不一致之处。

### 第三步：输出差异报告

生成结构化的差异报告，格式如下：

```
# 文档一致性检查报告

## 📄 [文档文件名]

### 总体状态：✅ 一致 / ⚠️ 有差异 / ❌ 严重过时

### 差异列表

#### 1. [差异标题]
- **严重程度**：错误 / 过时 / 缺失 / 多余
- **文档原文**：> 引用文档中的相关段落
- **代码实际情况**：描述代码中的实际行为
- **建议操作**：说明应如何修改文档

---

## 📊 汇总

| 文档 | 状态 | 错误 | 过时 | 缺失 | 多余 |
|------|------|------|------|------|------|
| ... | ... | ... | ... | ... | ... |

**总计**：X 份文档，Y 处差异
```

差异严重程度说明：
- **错误**：文档描述与代码完全矛盾
- **过时**：文档内容不再反映当前代码
- **缺失**：代码中有重要变更但文档未提及
- **多余**：文档描述的功能已被删除

## 注意事项

- **只读模式**：此技能不使用 Edit 或 Write 工具，不修改任何文件。
- **保守判断**：拿不准的差异标注为"待确认"，让用户自行判断。
- **聚焦实质**：忽略纯格式差异，聚焦内容准确性。
- **如需更新**：报告完成后，提示用户可使用 `/doc-sync:sync` 自动更新文档。
